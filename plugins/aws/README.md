AWS provides implementation workflows and MCP-backed guidance for architecture, operations, and AI workloads.

## What It Can Do
- Guides AWS architecture decisions for OpenAI-powered and general cloud workloads.
- Covers security, deployment, operations, and troubleshooting patterns.
- Connects to AWS MCP for account-aware guidance and workflow support.

## Why Use It
- Helps you pick practical AWS patterns without reinventing cloud architecture each time.
- Improves reliability and cost control for production AI systems.
- Turns vague cloud questions into concrete next actions.

## How It Works
1. Start with AWS workflow skills for architecture, security, and operations.
2. Use the AWS MCP integration when you need live account context or API-backed actions.
3. Apply recommendations directly in your infrastructure and app repos.

## Quick Examples
- `Design an AWS architecture for an OpenAI-powered API`
- `Set up secure secret management for OpenAI API keys on AWS`
- `Troubleshoot latency and cost for OpenAI workloads on AWS`

## Requirements
- AWS credentials and account access for live MCP-backed workflows.
- `uvx` available locally for the configured AWS MCP proxy command.
