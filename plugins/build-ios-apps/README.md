## What It Can Do
- Builds and refactors SwiftUI features with modern iOS patterns.
- Audits SwiftUI performance and identifies rendering bottlenecks.
- Supports simulator-driven debugging workflows for runtime issues.

## Why Use It
- Combines architecture guidance and debugging workflows in one place.
- Helps you ship cleaner SwiftUI code with fewer regressions.
- Speeds up diagnosis of UI and runtime issues on simulator builds.

## How It Works
1. Use the SwiftUI-focused skills for implementation and refactoring guidance.
2. Run iOS debugger workflows to build, launch, and inspect app behavior on simulator.
3. Iterate with targeted fixes, then verify behavior in the same workflow.

## Quick Examples

```
Build or refactor SwiftUI UI, audit performance, adopt modern iOS APIs, or debug an app on a simulator
```

## Requirements
- A macOS/Xcode environment with iOS simulator support.
- An iOS project in your workspace.
