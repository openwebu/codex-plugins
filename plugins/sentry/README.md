Sentry helps you inspect production issues and events to prioritize high-impact fixes.

## Top Skills
- `sentry`

## What It Can Do
- Inspects recent Sentry issues and events from your projects.
- Summarizes production error patterns for triage and prioritization.
- Supports read-only observability workflows focused on debugging decisions.

## Why Use It
- Cuts triage time by surfacing the most impactful error signals.
- Improves incident response with clear summaries and context.
- Keeps debugging grounded in real production telemetry.

## How It Works
1. Provide Sentry access credentials.
2. Run issue/event inspection workflows through the Sentry skill path.
3. Use findings to prioritize fixes in your repository backlog.

```
Inspect recent Sentry issues and summarize the top production errors
```

## Requirements
- `SENTRY_AUTH_TOKEN` available for Sentry API access.
